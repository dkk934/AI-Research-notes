# AI Research Notes

## Description

This repository contains curated research notes, summaries, and practical experiments related to artificial intelligence (AI) and machine learning (ML). It is intended as a personal learning space to collect important ideas, references to papers, simple code snippets, and experiment logs while studying AI topics.

## Reason for this repository

Gaining basic knowledge about AI, machine learning, and related subfields; building a searchable, organized notebook of concepts, papers, and hands-on examples; and creating a reference that can be revisited and expanded over time.

Key goals:
- Learn foundational AI/ML concepts and algorithms
- Keep summaries of important research papers
- Record small experiments and reproducible code snippets
- Organize links and references for future study

## Suggested file / folder structure

- README.md                         - This file
- notes/                            - Text notes and concept summaries (organized by topic)
  - fundamentals.md
  - deep-learning.md
  - probabilistic-models.md
- papers/                            - Paper summaries and bib entries
  - 2025-deep-learning-survey.md
- code/                              - Small reproducible example scripts and notebooks
  - experiments/                     - Experiment code and results
  - utils/                            - Helper scripts
- datasets/                          - Pointers or small example datasets (do not store large datasets here)
- experiments/                       - Logs, run notes, and results
- references/                        - Links, books, and further reading
- slides/                            - Presentation slides or short overviews

## How to use

- Read notes in the `notes/` folder for quick conceptual overviews.
- Check `papers/` for short summaries and takeaways from research papers.
- Run examples from `code/` to reproduce small experiments or toy models.
- Add new files following the existing organization: topic-based notes in `notes/`, paper digests in `papers/`, and code in `code/`.
